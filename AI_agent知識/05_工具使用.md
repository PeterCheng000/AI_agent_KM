# 05 — 工具使用：Agent 怎麼和外部世界互動？

> **核心問題**：LLM 只能輸出文字。要讓 Agent 真正「做事」，必須能呼叫外部工具。這件事是怎麼運作的？

---

## 工具使用的本質

沒有工具的 Agent：
```
知道「台北現在下雨」← 只能靠訓練資料，可能過時
```

有工具的 Agent：
```
呼叫 weather_api("台北") → 即時獲得當前天氣 → 回答
```

**工具讓 Agent 從「依賴記憶」變成「主動獲取最新資訊和執行操作」。**

---

## 演化：從 ReAct 到 Function Calling

### 早期（ReAct 暴力法）

```
模型輸出純文字：
"Action: search('台積電股價')"

程式解析：
import re
match = re.search(r"Action: (\w+)\('(.+)'\)", output)
tool_name = match.group(1)  # "search"
tool_args = match.group(2)  # "台積電股價"
```

**問題**：
- 模型輸出格式不穩定，regex 容易失敗
- 模型可能幻覺出不存在的工具
- 沒有型別檢查，容易傳錯參數

---

### 現在（Function Calling / Tool Use）

模型直接輸出結構化 JSON：

```json
{
  "type": "tool_use",
  "name": "web_search",
  "input": {
    "query": "台積電最新股價",
    "max_results": 5
  }
}
```

**優點**：
- 格式保證正確（模型被訓練成嚴格遵守 schema）
- 有型別驗證
- 參數名稱明確，不會猜錯

---

## 工具的定義方式

每個工具都需要告訴模型：它叫什麼、能做什麼、需要什麼參數。

```python
tools = [
    {
        "name": "web_search",
        "description": "搜尋網路上的即時資訊。適合查詢新聞、股價、天氣等即時數據。",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "搜尋關鍵字"
                },
                "max_results": {
                    "type": "integer",
                    "description": "回傳結果數量，預設 5",
                    "default": 5
                }
            },
            "required": ["query"]
        }
    }
]
```

> **重點**：`description` 非常重要。模型靠這段文字決定「要不要用這個工具、什麼情況下用」。描述寫得好不好，直接影響 Agent 的行為品質。

## 工具呼叫的完整流程

```
① 用戶提問：「台積電今天股價多少？」
         │
         ▼
② 模型判斷：需要工具
  輸出：{ "name": "web_search", "input": { "query": "台積電股價" } }
         │
         ▼
③ 程式執行工具
  result = web_search("台積電股價")
  → 回傳：「台積電今日收盤 xxx 元，漲 x.x%」
         │
         ▼
④ 把結果傳回給模型（作為 Observation）
  [tool_result]: 「台積電今日收盤 xxx 元，漲 x.x%」
         │
         ▼
⑤ 模型整合資訊，生成最終回答
  「台積電今天收盤價為 xxx 元，上漲 x.x%。」
```

## 工具的四個層次

```
┌────────────────────────────────────────────┐
│  資訊類（查詢，不改變世界）                │
│  Web Search、Database Read、File Read      │
│  → 最安全，可以自動執行                    │
├────────────────────────────────────────────┤
│  計算類（確定性輸出）                      │
│  Calculator、Code Execution、Data Analysis │
│  → 安全，但 Code Execution 需要沙箱        │
├────────────────────────────────────────────┤
│  行動類（改變世界，有副作用）              │
│  Send Email、Write File、API POST          │
│  → 高風險，建議要人類確認                  │
├────────────────────────────────────────────┤
│  Agent 類（呼叫另一個 Agent）              │
│  Sub-agent、Specialized Agent              │
│  → 複雜，需要設計好錯誤處理               │
└────────────────────────────────────────────┘
         風險 ↑，越往下越需要謹慎
```

## 常見工具清單

### 資訊類
| 工具 | 用途 |
|------|------|
| Web Search | 搜尋即時網路資訊 |
| Web Fetch | 讀取特定網頁完整內容 |
| Database Query | 查詢結構化資料 |
| File Read | 讀取本地或雲端檔案 |
| Calendar Read | 讀取行事曆 |

### 計算類
| 工具 | 用途 |
|------|------|
| Calculator | 精確數學計算 |
| Code Interpreter | 執行 Python 等程式碼 |
| Data Analysis | 分析 CSV / DataFrame |
| Image Analysis | 分析圖片內容 |

### 行動類
| 工具 | 用途 |
|------|------|
| Send Email | 發送電子郵件 |
| Create File | 建立文件 |
| Database Write | 寫入資料庫 |
| API Call | 呼叫外部服務 |
| Browser Control | 控制瀏覽器行為 |

## 多工具協作範例

**任務**：「幫我分析競爭對手的最新動態，整理成報告寄給我」

```
Step 1: web_search("競爭對手 A 最新新聞")
        → Observation: 找到 5 篇文章

Step 2: web_fetch(url_1), web_fetch(url_2) ...
        → Observation: 取得完整文章內容

Step 3: code_interpreter("""
          分析文章，提取：公司名稱、事件類型、日期、影響
          輸出 JSON
        """)
        → Observation: 結構化的競品資訊

Step 4: create_file("競品分析報告.md", content=report)
        → Observation: 檔案已建立

Step 5: send_email(to="user@company.com", 
                   subject="競品分析報告",
                   attachment="競品分析報告.md")
        → Observation: 郵件已發送

Final Answer: 報告已生成並寄送給您。
```

## 設計好工具的三個原則

### 原則一：描述要清楚說明「什麼時候用」

```python
# ❌ 差的描述
"description": "搜尋資訊"

# ✅ 好的描述
"description": """搜尋網路上的即時資訊。
適合：最新新聞、即時股價、近期事件、不確定是否過時的資訊。
不適合：需要登入才能看到的內容、需要計算的問題。"""
```

### 原則二：工具職責單一

```
❌ 一個工具做太多事：
  "search_and_summarize_and_save"

✅ 每個工具只做一件事：
  "web_search" + "summarize" + "save_file"
```

### 原則三：輸出格式要結構化

```python
# ❌ 回傳純文字
return "找到台積電股價是 xxx 元"

# ✅ 回傳結構化資料
return {
    "company": "台積電",
    "price": xxx,
    "change_percent": x.x,
    "timestamp": "2025-xx-xx xx:xx"
}
```
