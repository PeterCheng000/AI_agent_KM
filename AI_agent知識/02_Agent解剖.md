# 02 — Agent 解剖：一個 Agent 內部怎麼運作？

> **核心問題**：不管哪個階段，Agent 內部永遠是哪些模組在運作？

---

## Agent 的四個核心模組

```
┌─────────────────────────────────────────┐
│              AI Agent                   │
│                                         │
│  ┌──────────┐      ┌──────────┐        │
│  │  規劃    │      │  記憶    │        │
│  │ Planning │      │  Memory  │        │
│  └────┬─────┘      └────┬─────┘        │
│       │                  │              │
│  ┌────▼──────────────────▼────┐        │
│  │          LLM Core          │        │
│  └────────────────┬───────────┘        │
│                   │                     │
│  ┌────────────────▼───────────┐        │
│  │      工具 / Actions        │        │
│  │   Tool Use / Execution     │        │
│  └────────────────────────────┘        │
└─────────────────────────────────────────┘
```

這四個模組的成熟度，決定了 Agent 能處理的任務複雜度。

---

## 模組一：規劃（Planning）

**核心問題**：接下來要做什麼步驟？

規劃讓 Agent 從「被動回答」變成「主動分解問題」。

### 主要策略

| 策略 | 特性 | 適合場景 |
|------|------|---------|
| ReAct | 邊想邊做，線性推進 | 大多數通用任務 |
| Chain of Thought | 深度推理，不行動 | 數學、邏輯推導 |
| Plan-and-Execute | 先完整規劃，再逐步執行 | 長任務、步驟多 |
| Tree of Thought | 探索多條路徑，選最好的 | 創意、需要評估多方案 |
| Reflexion | 執行後自我反思，重試 | 需要高準確率的任務 |

詳見：[03 — 規劃策略](03_規劃策略.md)

## 模組二：記憶（Memory）

**核心問題**：我知道什麼？

### Context Window 的本質

很多人以為 context window 只是「記憶長度」，但它其實是 **Agent 的完整工作空間**：

```
┌──────────────── Context Window ─────────────────┐
│                                                  │
│  [System Prompt]   你是一個助理，有以下工具...   │
│  [Long-term Memory] 用戶叫 Alice，是工程師...    │
│  [Tool Results]    search() 回傳：「...」        │
│  [Conversation]    User: ... / Assistant: ...   │
│  [Current Task]    現在要回答：「比較兩家...」   │
│                                                  │
│  ← 模型「看到」的全部就是這些 →                  │
└──────────────────────────────────────────────────┘
```

### 四層記憶架構

```
Layer 1：In-Context Memory（最快）
  當前對話直接放在 context window 裡
  限制：context 有上限，超過就截斷

Layer 2：Conversation Summary（省空間）
  舊對話自動用 LLM 壓縮成幾句話
  效果：省大量 token，但細節會損失

Layer 3：Entity Memory（精確）
  抽取重要事實，結構化儲存
  例：{ "用戶": "Alice", "職業": "工程師", "偏好": "簡短回答" }

Layer 4：Vector Memory（語意搜尋）
  重要對話片段向量化存入資料庫
  用途：找歷史相關記憶，跨對話使用
```

詳見：[04 — 記憶管理](04_記憶管理.md)

## 模組三：工具（Tools）

**核心問題**：我能做什麼？

工具讓 LLM 從「知道」變成「能做到」。

### 工具的四個層次

```
資訊類（查詢，不改變世界）
  Web Search、Database Query、File Read

計算類（確定性輸出）
  Calculator、Code Execution、Data Analysis

行動類（改變世界，有副作用）
  Send Email、Create File、API Call

Agent 類（呼叫另一個 Agent）
  Sub-agent、Specialized Agent
```

> 工具越往下，風險越高，越需要人類確認。

詳見：[05 — 工具使用](05_工具使用.md)

## 模組四：行動與執行（Action）

**核心問題**：執行後，結果是什麼？

行動是把規劃變成真實世界的改變。每次行動後，Agent 會觀察結果（Observation），再決定下一步。

```
規劃 → 行動 → 觀察結果 → 更新認知 → 規劃下一步
  ↑                                       │
  └───────────────────────────────────────┘
               （迭代循環）
```

### 關鍵設計原則

**可逆優先**：能用可逆操作就不用不可逆操作。
```
讀取 > 草稿 > 預覽 > 確認後執行
```

**最小權限**：Agent 只獲得完成任務所需的最小工具集，降低意外風險。

## Agent 執行的完整流程

```
使用者輸入
    │
    ▼
┌─────────────────────────────────────┐
│  1. 理解意圖                        │
│     用戶真正想要什麼？               │
└──────────────────┬──────────────────┘
                   │
┌──────────────────▼──────────────────┐
│  2. 從記憶取得上下文                │
│     過去說了什麼？有哪些相關資訊？   │
└──────────────────┬──────────────────┘
                   │
┌──────────────────▼──────────────────┐
│  3. 規劃步驟                        │
│     需要做哪些事？順序是什麼？       │
└──────────────────┬──────────────────┘
                   │
         ┌─────────▼─────────┐
         │   需要工具嗎？     │
         └────┬──────────┬───┘
             Yes          No
              │            │
┌─────────────▼──┐   ┌─────▼──────────┐
│ 呼叫工具       │   │ 直接用 LLM 回答 │
│ 取得 Observation│   └────────────────┘
└────────┬───────┘
         │
┌────────▼───────────────────────────┐
│  4. 判斷是否完成                   │
│     達到目標了嗎？還是需要繼續迭代？│
└────────┬───────────────────────────┘
         │
    ┌────▼────┐
    │ 完成了？ │
    └──┬──┬───┘
      Yes  No
       │    └──────── 回到步驟 3
       ▼
    輸出給用戶
```

---

## 一句話總結四個模組

| 模組 | 一句話 | 類比 |
|------|--------|------|
| 規劃 | 決定要做什麼、怎麼做 | 大腦的前額葉 |
| 記憶 | 知道什麼、記得什麼 | 大腦的記憶系統 |
| 工具 | 能做什麼、用什麼做 | 雙手 + 工具箱 |
| 行動 | 真正去做，觀察結果 | 身體的執行 |
