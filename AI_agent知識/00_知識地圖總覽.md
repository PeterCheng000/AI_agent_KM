# LLM AI Agent 知識中心

> **核心主線**：「怎麼讓 AI 從『會說話』變成『會做事』？」

---

## 知識地圖

```
為什麼需要 Agent？
LLM 只會說，不會做
        │
        ▼
怎麼讓它「做」？
Chain → ReAct → Tool Use → Multi-Agent
（自動化）（自主）（可靠）  （規模化）
        │
        ▼
Agent 內部怎麼運作？
規劃 × 記憶 × 工具 × 行動
        │
        ▼
上線會遇到什麼？
不可靠 → 結構化輸出 + 護欄
太貴   → 快取 + 模型分級
不可控 → Human-in-the-loop
        │
        ▼
還沒解決的問題
Memory 一致性 ／ Evaluation 方法論
```

## 關鍵術語速查

| 術語 | 一句話解釋 |
|------|-----------|
| LLM | 大型語言模型，AI 的思考核心 |
| Agent | 能自主規劃、使用工具、迭代執行的 AI |
| Chain | 把多個 LLM 呼叫串成固定流程 |
| ReAct | 讓模型交替「思考」和「行動」的框架 |
| Tool Use | 模型以結構化方式呼叫外部工具 |
| RAG | 讓 Agent 從外部知識庫檢索資料再回答 |
| Multi-Agent | 多個 Agent 分工協作處理複雜任務 |
| Context Window | 模型一次能「看到」的完整工作空間 |
| Embedding | 把文字轉成向量，用於語意搜尋 |
| RLHF | 用人類回饋訓練模型對齊人類意圖 |
